Import projects

  • Click Import to import the implementation of a retail solution into your workspace. The solution contains the following:
    • Application BTM_RETAIL_APP is the entry point.
      Main_Process_Order_Flow.msgflow routes an incoming message to one of BTM_RETAIL_ORDER_ITEM_APP, BTM_RETAIL_PURCHASE_ITEM_APP, BTM_RETAIL_STOCK_CHECK_APP for further processing depending on the type of request.
    • Application BTM_RETAIL_ORDER_ITEM_APP is invoked if the type of request is "orderItem". Order_Item_Request_Flow.msgflow receives the request. Order_Item_Despatch_Flow.msgflow processes the request and Order_Item_Response_Flow.msgflow sends a confirmation.
    • Application BTM_RETAIL_PURCHASE_ITEM_APP is invoked if the type of request is "purchaseItem". Purchase_Item_Request_Flow.msgflow receives the request. Process_Payment_Flow.msgflow processes the request and Purchase_Item_Response_Flow.msgflow sends a confirmation.
    • Application BTM_RETAIL_STOCK_CHECK_APP is invoked if the type of request is "stockCheck". Stock_Check_Request_Flow.msgflow receives the request. Check_Stock_Flow.msgflow processes the request and Stock_Check_Response_Flow.msgflow sends a confirmation.
    • Shared library BTM_RETAIL_SHARED_LIBRARY contains the message model used by the solution.
  • All message flows have monitoring events configured on a number of their message nodes.
    For this tutorial, all monitoring events use $Root/XMLNSC/btm_retail/customerNumber as the global correlator. To see an example, open Main_Process_Order_Flow.msgflow, select "Receive Retail Request" node, click the Monitoring tab in the Properties view.
  • For simplicity, this tutorial will only monitor the messages processed by BTM_RETAIL_ORDER_ITEM_APP. Once you have completed this tutorial, you can explorer further by creating more business transaction definitions using the message flows in the other applications.

Create queues

  1. Create the default IBM Integration Bus queues
    1. Open the IBM Integration Console.
    2. In the console window, type
      server\sample\wmq\iib_createqueues qmgr
      where
      qmgr is the queue manager name
  2. Create the queues used by the MQ nodes in this tutorial
    1. In the console window, type
      runmqsc qmgr
      where
      qmgr is the queue manager name
    2. Copy and paste the following into the console window to create the queues:
      define qlocal(BTM_RETAIL_INQ) replace
      define qlocal(BTM_RETAIL_OUTQ) replace
      define qlocal(BTM_RETAIL_FAILUREQ) replace
      define qlocal(BTM_RETAIL_UNKNOWN_ORDER_OUTQ) replace
      define qlocal(BTM_RETAIL_ORDER_ITEM_INQ) replace
      define qlocal(BTM_RETAIL_ORDER_ITEM_OUTQ) replace
      define qlocal(BTM_RETAIL_ORDER_ITEM_DESPATCH_INQ) replace
      define qlocal(BTM_RETAIL_ORDER_ITEM_DESPATCH_OUTQ) replace
      define qlocal(BTM_RETAIL_STOCK_CHECK_INQ) replace
      define qlocal(BTM_RETAIL_STOCK_CHECK_OUTQ) replace
      define qlocal(BTM_RETAIL_PROCESS_STOCK_CHECK_INQ) replace
      define qlocal(BTM_RETAIL_PROCESS_STOCK_CHECK_OUTQ) replace
      define qlocal(BTM_RETAIL_PURCHASE_ITEM_INQ) replace
      define qlocal(BTM_RETAIL_PURCHASE_ITEM_OUTQ) replace
      define qlocal(BTM_RETAIL_PROCESS_PAYMENT_INQ) replace
      define qlocal(BTM_RETAIL_PROCESS_PAYMENT_OUTQ) replace
  3. Associate the default integration node with the queue manager
    1. In the console window, stop the default integration node.
      mqsistop -i TESTNODE_userid
    2. Associate the queue manager with the default integration node, TESTNODE_userid
      mqsichangebroker TESTNODE_userid -q qmgr
      where
      qmgr is the queue manager name
    3. Start the default integration node.
      mqsistart TESTNODE_userid

Deploy projects

  • Drag and drop BTM_RETAIL_ORDER_ITEM_APP and BTM_RETAIL_SHARED_LIBRARY onto an integration server of TESTNODE_userid.
  • Note: BTM_RETAIL_APP will be deployed in a later step.

Create a business transaction definition

  1. Right-click on TESTNODE_userid in the Integration Nodes view and select Start Web User Interface.
  2. Create a business transaction definition
    In the web user interface, expand Business -> Business Transactions. Select Create from the menu to create a new business transaction. Enter RetailOrderBTD. Click OK.
  3. Add message flows to the business transaction definition
    1. In the Define tab of RetailOrderBTD, click Add flow to select the message flows that make up this business transaction.
    2. Inside the popup, click Select application, then click the dropdown and select BTM_RETAIL_ORDER_ITEM_APP. Click Add to add all the message flows in this application to RetailOrderBTD.
    3. Observe that Order_Item_Request_Flow.msgflow, Order_Item_Despatch_Flow.msgflow, and Order_Item_Response_Flow.msgflow are added to the Business Transaction section.
      The warning icon indicates that no start and end events have been defined for this business transaction.
  4. Select the monitoring events that indicate the start, end, and failure of the business transaction definition. In the Business Transaction Event Definitions section,
    1. Expand Order_Item_Request_Flow.msgflow, flag the "Order Item Request Received" event as the Start event.
    2. Expand Order_Item_Despatch_Flow.msgflow, flag the "Item Despatched" event as the Progress event.
    3. Expand Order_Item_Response_Flow.msgflow, flag the "Confirm Item Ordered" event as the End event.
  5. Save RetailOrderBTD.

Prepare database

  1. Run this script to create the database MBRECORD and monitoring events tables:
    IIBInstallDir/server/ddl/db/DataCaptureSchema.sql
    where
    IIBInstallDir is the installed location of IBM Integration Bus Version 10.0.0.3, and
    db is either db2 or oracle.
    Note:To customize the database or schema name used by business transaction monitoring, edit DataCaptureSchema.sql before running it.
  2. Run this script to create the tables for recording and storing business transaction data:
    IIBInstallDir/server/ddl/db/BusinessCaptureSchema.sql
    Note: If you modified the name of the database in the previous step, you'll need to update the database name in BusinessCaptureSchema.sql before running it.
  3. Define a ODBC data source name (DSN), MBRECORD, for the database MBRECORD.
    Follow the link Enabling ODBC connections to the databases in the Find out more section below for instructions on configuring an ODBC data source for your operating system.
    Note: If you modified the name of the database in the previous steps, use the same name for your DSN name in this step.
  4. Set a user id and password for the integration node to use when connecting to the database.
    1. Open the IBM Integration Console.
    2. In the console window, type
      mqsisetdbparms TESTNODE_userid -n DSN -u id -p password
      where
      TESTNODE_userid is the default integration node
      DSN is the data source name created in the previous step
      id and password are the user id and password used to connect to the database MBRECORD
    3. Optional: Verify the user id and password for the database is set correctly by typing
      mqsireportdbdbparms TESTNODE_userid -n DSN -u id -p password
  5. Restart the integration node.
    In the console window, type
    mqsistop -i TESTNODE_userid
    After the default integration node is stopped, type
    mqsistart TESTNODE_userid
  6. Optional: Test the connection to the database MBRECORD.
    In the console window, type
    mqsicvp TESTNODE_userid -n DSN

Configure the database to be used for the business transaction definition

  1. Open the Web User Interface.
  2. Expand Business -> Business Transactions. Select RetailOrderBTD.
  3. Select the Configure tab. Note that RetailOrderBTD is not yet configured with a data source.
  4. Click Click here to specify a data source name. Select the data source name MBRECORD from the dropdown.
  5. Click Save to save the datasource name. Then click Resume editing of business transaction RetailOrderBTD to return to the Configure tab.
    Note: MBRecord will be used to record all the business transactions for this integration node.

Start monitoring the message flows

  1. In the web user interface, expand Servers -> server -> Applications -> BTM_RETAIL_ORDER_ITEM_APP
    where
    server is the integration server where BTM_RETAIL_ORDER_ITEM_APP is deployed.
  2. Select Start flow monitoring from the menu of BTM_RETAIL_ORDER_ITEM_APP.

Run Main_Process_Order_Flow in the flow exerciser

  1. Open Main_Process_Order_Flow. Click the Flow Exerciser icon to start recording the message path through the flow.
  2. Click the Send Message icon to select a message to send to the flow. Choose Cookie Monster's Order. Note the "requestType" is "orderItem" and "customerNumber" is "5555". Click Send.
  3. After the response message is received, the message path is highlighted on the message flow. You may click on any connection to see the message that passed through that connection.
  4. Click the Send Message icon again. Choose Panda Po's Order. Note the "requestType" is "orderItem" and "customerNumber" is "5670". Click Send.
  5. Click the Send Message icon again. Choose Winnie the Pooh's Order. Note the "requestType" is "orderItem" and "customerNumber" is "4785". Click Send.

All 3 transactions were processed by the message flows in BTM_RETAIL_ORDER_ITEM_APP and captured by RetailOrderBTD. You will now be able to view the results in the Web User Interface.

View the instances of RetailOrderBTD

  1. Open the Web User Interface.
  2. Expand Business -> Business Transactions. Select RetailOrderBTD. Select the View tab.
  3. Click Refresh in the All instances of RetailOrderBTD section. 3 instances, identified by the global transaction ID "customerNumber", will be shown. All instances have the status of "Ended".
  4. Click on an instance. The 3 associated events, "Order Item Request Received", "Item Despatched", "Confirm Item Ordered", and their corresponding details will be shown in the Events section.
  5. Optionally, to show only the instances with a specified global transaction ID, or those that have a particular status, click Filter in the All instances of RetailOrderBTD section to specify the filtering criteria.
